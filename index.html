2. Data We Collect

None. ArchiveSorter does not collect, log, transmit, or store any personal data, usage data, analytics, crash reports, advertising identifiers, or device identifiers on any server operated by the Developer.

The Developer does not operate any backend service for this app.

3. On-Device Processing

All document scanning, optical character recognition (OCR), classification, and language model inference is performed locally on your device using:

  • Apple’s Vision framework (OCR)
  • Apple’s Foundation Models framework (on-device language model)
  • Apple’s MLX framework (on-device language model inference)

Your documents, their contents, and any text extracted from them never leave your device through any feature provided by the Developer.

4. Optional Network Connections

Two optional features may cause network traffic. Both are initiated by you and are not part of automatic data collection:

4.1 LM Studio (local network, optional)

You may configure the app to use a local LM Studio server that you run yourself on your own computer or local network. When you enable and use this feature, document text is sent over your local network to the LM Studio server you control. No data leaves your local network. The Developer does not operate or have access to this server.

4.2 Hugging Face Model Downloads (optional)

When you choose to download an MLX language model, the app fetches the model files directly from the Hugging Face Hub (huggingface.co). Only the model files are downloaded; no document content, personal data, or usage information is sent to Hugging Face by the app. Standard HTTP request metadata (such as your IP address) is visible to Hugging Face, governed by their own privacy policy.

5. Permissions the App Requests

  • Camera — only when you scan a document using the in-app document scanner. Captured images are processed on-device and stored only in the folder you selected as your library.
  • Face ID / Touch ID — only to unlock the app, if you enable App Lock. Biometric data is handled by iOS and never seen by the app or the Developer.
  • Local Network — only if you configure the optional LM Studio integration described above.
  • File access — the app reads and writes only inside the folder you explicitly select as your library.

6. Data Storage

All data created or processed by the app (scanned documents, classifications, settings, the app’s on-device index) is stored exclusively on your device or in a folder you choose, which may include a cloud-synchronised folder you have set up yourself (such as iCloud Drive, Dropbox, OneDrive, or Google Drive). The Developer does not control, access, or process data stored in such third-party cloud folders. Their handling is governed by the respective provider’s privacy policy.

7. Children’s Privacy

ArchiveSorter is not directed at children under 13. The app does not knowingly collect any data from children, as it does not collect data from anyone.

8. Third-Party Services

The app does not include any third-party analytics, advertising, crash reporting, or tracking SDKs. The only optional third-party service the app may communicate with is Hugging Face Hub, when you download a language model.

9. Your Rights

Because the Developer does not collect or store any personal data about you, there is no personal data to access, rectify, delete, or port. All data created in the app stays on your device or in your chosen folder, where you have full control and can delete it at any time.

10. Changes to This Policy

If this policy changes, the “Last updated” date at the top of this page will reflect the change. Please check back occasionally.

2. Données collectées

Aucune. ArchiveSorter ne collecte, n’enregistre, ne transmet ni ne stocke aucune donnée personnelle, donnée d’utilisation, donnée analytique, rapport de plantage, identifiant publicitaire ou identifiant d’appareil sur un serveur exploité par le Développeur.

Le Développeur n’exploite aucun service backend pour cette application.

3. Traitement sur l’appareil

La numérisation des documents, la reconnaissance optique de caractères (OCR), la classification et l’inférence du modèle linguistique sont entièrement réalisées localement sur votre appareil à l’aide de :

  • Le framework Vision d’Apple (OCR)
  • Le framework Foundation Models d’Apple (modèle linguistique sur l’appareil)
  • Le framework MLX d’Apple (inférence de modèle linguistique sur l’appareil)

Vos documents, leur contenu et tout texte extrait ne quittent jamais votre appareil par une fonctionnalité fournie par le Développeur.

4. Connexions réseau optionnelles

Deux fonctionnalités optionnelles peuvent générer du trafic réseau. Les deux sont initiées par vous et ne font pas partie d’une collecte automatique de données :

4.1 LM Studio (réseau local, optionnel)

Vous pouvez configurer l’application pour utiliser un serveur LM Studio local que vous exécutez vous-même sur votre ordinateur ou votre réseau local. Lorsque vous activez cette fonctionnalité, le texte du document est envoyé via votre réseau local au serveur LM Studio que vous contrôlez. Aucune donnée ne quitte votre réseau local. Le Développeur n’exploite pas ce serveur et n’y a pas accès.

4.2 Téléchargements de modèles Hugging Face (optionnel)

Lorsque vous choisissez de télécharger un modèle linguistique MLX, l’application récupère les fichiers du modèle directement depuis Hugging Face Hub (huggingface.co). Seuls les fichiers du modèle sont téléchargés ; aucun contenu de document, donnée personnelle ou information d’utilisation n’est envoyé à Hugging Face par l’application. Les métadonnées HTTP standard (comme votre adresse IP) sont visibles par Hugging Face et soumises à leur propre politique de confidentialité.

5. Autorisations demandées par l’application

  • Caméra — uniquement lors de la numérisation d’un document avec le scanner intégré. Les images capturées sont traitées sur l’appareil et stockées uniquement dans le dossier que vous avez sélectionné comme bibliothèque.
  • Face ID / Touch ID — uniquement pour déverrouiller l’application si vous activez le verrouillage. Les données biométriques sont gérées par iOS et ne sont jamais accessibles à l’application ou au Développeur.
  • Réseau local — uniquement si vous configurez l’intégration optionnelle LM Studio décrite ci-dessus.
  • Accès aux fichiers — l’application lit et écrit uniquement dans le dossier que vous sélectionnez explicitement comme bibliothèque.

6. Stockage des données

Toutes les données créées ou traitées par l’application (documents numérisés, classifications, paramètres, index local) sont stockées exclusivement sur votre appareil ou dans un dossier de votre choix, qui peut inclure un dossier synchronisé dans le cloud (iCloud Drive, Dropbox, OneDrive ou Google Drive). Le Développeur ne contrôle pas, n’accède pas et ne traite pas les données stockées dans ces dossiers cloud tiers. Leur gestion est régie par la politique de confidentialité du fournisseur concerné.

7. Protection des mineurs

ArchiveSorter ne s’adresse pas aux enfants de moins de 13 ans. L’application ne collecte sciemment aucune donnée d’enfants, car elle ne collecte aucune donnée de qui que ce soit.

8. Services tiers

L’application n’intègre aucun SDK d’analyse, de publicité, de rapport de plantage ou de suivi tiers. Le seul service tiers optionnel avec lequel l’application peut communiquer est Hugging Face Hub, lors du téléchargement d’un modèle linguistique.

9. Vos droits

Étant donné que le Développeur ne collecte ni ne stocke aucune donnée personnelle vous concernant, il n’existe aucune donnée personnelle à consulter, rectifier, supprimer ou transférer. Toutes les données créées dans l’application restent sur votre appareil ou dans votre dossier choisi, où vous avez un contrôle total et pouvez les supprimer à tout moment.
